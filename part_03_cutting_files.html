Порізка макета, організація робочого простору.

1. Порізка макета.

Запустіть програму Adobe Photoshop і відкрийте макет сайту з продажу футболок (jpg).

Для вирізання елементів використовується інструмент "Скальпель" (гаряча клавіша C).

При вирізанні слайсів одразу задавайте їм імена (подвійний клік по слайсу чи клік із зажатим Ctrl).

Якщо в макеті є подібні елементи - виріжте один, дайте йому ім'я, далі - копіюйте цей слайс, тримаючи клавішу Alt.

В першу чергу вирізайте фотографії, іконки, логотипи, відключіть всі фони, щоб тло було "прозоре" (шахматний фон). Збережіть ці зображення, включіть всі фони і вимкніть шари з текстом та вирізаними зображеннями, розмітьте і збережіть всі фони.

Щоб видалити всі слайси - можна скористатися пунктом меню View > Remove slices або Просмотр > Удалить фрагменты.

2. Збереження зображень, типи файлів.

Для збереження вирізаних слайсів скористайтеся пунктом меню чи гарячими клавішами:

File > Save For Web...
Файл > Сохранить для Web и устройств...
Ctrl + Alt + Shift + S

У цьому вікні потрібно підібрати формати файлів, якість стиснення для кожного слайса.

Для зручності - виділяйте декілька подібних слайсів, утримуючи Shift.

Формати файлів:

JPG - для фотографій, великих фонових зображень;

PNG - для елементів дизайну, іконок, всіх елементів з прозорістю;

GIF - для дуже маленьких елементів з малою кількістю кольорів.

Для формату JPG вибирайте якість 60-70 для невеликих, неякісних зображень, розпливчастих, блідих фонів, 80-90 - для чітких, крупних, хороших зображень.

Червоні кольори більше за інших створюють "артифакти" в форматі стиснення JPEG, тому при значних кількостях червоного кольору та різких переходів до контрасних до нього кольорів бажано підвищити якість збереження.

Зображення 2.1. Артифакти JPEG.

Імена файлам потрібно давати з розумом, таким чином, щоб подібні файли знаходилися поруч в списку файлів. Для цього використовуються префікси.

Зображення 2.2. Іменування файлів, зліва - невірне.

2. Організація робочого простору.

З часом у вас буде все більше і більше проектів, різноманітних робіт.

Щоб дотримуватися якогось порядку, не перетворити робочий стіл і вінчестер на хаотичне нагромадження файлів, прислухайтесь до порад.

Створіть папку work на диску D:\ (чи будь-якому іншому, не на C:\), для MacOS та Linux - в документах, домашньому каталозі.

Створіть папку frontend в папці work (в англійській мові немає поняття веб-верстки, layouting стосується поліграфії, зазвичай пишуть щось типу html-coding).

На FTP створіть папку зі своїм іменем, прізвиськом і вважайте її аналогом папки verstka.

2. Формат архівів.

Завжди добавляйте дату до імені архіву.

Завжди давайте зрозумілі назви архівів, не плодіть файли html.zip, html(2).zip, html(3).zip...

projectName_date.zip
futbolki_20151110.zip
futbolki_20151110a.zip
futbolki_20151110b.zip
futbolki_20151110c.zip

Для архівування використовуємо лише формат ZIP, всі сучасні операційні системи мають інструменти для розпакування zip-архівів. Архіви в форматах RAR, 7z можуть не розпакуватися у замовника.

Коли я перевірю ваше домашнє завдання - то зроблю файл-репорт з описом ваших помилок чи похвалою, що все добре і правильно зроблено:

futbolki_20151110.txt

Якщо ви допустили якісь незначні помилки і розумієте їх, як їх виправити і уникнути в майбутньому - то нову, виправлену версію на сервер можна не заливати.

Якщо наробили купу помилок, то роботу необхідно переробити, виправити і заляти на сервер з новою датою архіва.

Домашнє завдання.

Результати домашніх завдань заляйте на ftp в свої папки.
